# ExecPlan Template

Use this template for all task execution plans in this repository.

Reference template path: `/Users/michaelwall/codex-live-bridge/PLANS.md`

## Rules

1. Keep each plan self-contained and executable by someone new to the repo.
2. Do not use fenced code blocks in plan files. Use indented command snippets.
3. Define non-obvious terms in plain language.
4. Use absolute repo file paths for all referenced files.
5. Include concrete commands, expected outputs, validation steps, and acceptance criteria.
6. Include all required sections listed below.

## Required Sections

1. `Title`
2. `Reference`
3. `Goal`
4. `Scope`
5. `Out of Scope`

10. `Acceptance Criteria`
11. `Risks & Mitigations`
12. `Progress`
13. `Surprises & Discoveries`
14. `Decision Log`
15. `Outcomes & Retrospective`
